בסוף <a href="http://www.gadial.net/2012/03/01/inner_product_spaces_geometry/">הפוסט האחרון שלי</a> על אלגברה לינארית הבטחתי שנדבר על מה שקורה כשלוקחים מרחב מכפלה פנימית ומתחילים לדבר על טרנספורמציות לינאריות מעליו. אני רוצה להתחיל עם הסוג הפשוט ביותר של טרנספורמציות לינאריות - פונקציונלים לינאריים.

תזכורת קצרה: {% equation %}V{% endequation %} הוא מרחב מכפלה פנימית אם הוא מרחב וקטורי מעל {% equation %}\mathbb{C}{% endequation %} שמוגדרת בו פונקציה {% equation %}\left\langle \cdot,\cdot\right\rangle :V\times V\to\mathbb{C}{% endequation %} שהיא לינארית במשתנה הראשון (כלומר, {% equation %}\left\langle \lambda x+y,z\right\rangle =\lambda\left\langle x,z\right\rangle +\left\langle y,z\right\rangle {% endequation %}), הרמיטית - {% equation %}\left\langle x,y\right\rangle =\overline{\left\langle y,x\right\rangle }{% endequation %} ("כמעט" סימטרית עד כדי הצמוד למעלה, שאת הסיבות לו הסברתי בעבר) וחיובית, כלומר {% equation %}\left\langle x,x\right\rangle \ge0{% endequation %} ושוויון הוא רק כש-{% equation %}x=0{% endequation %}. פונקציונל לינארי במקרה שלנו הוא פונקציה לינארית {% equation %}f:V\to\mathbb{C}{% endequation %}. הדמיון למכפלה פנימית די ברור - פונקציונל פועל על איבר אחד ואילו מכפלה פנימית על שניים, אבל שניהם מחוייבים ללינאריות מסוג כלשהו. אם המרחב {% equation %}V{% endequation %} הוא סוף-ממדי (ואני מתכנן לדבר רק על מרחבי מכפלה פנימית סוף ממדיים; מה שקורה במרחבים אינסוף מימדיים הוא נושא לסדרת פוסטים ארוכה אף יותר מהנוכחית), הדמיון הוא לחלוטין לא מקרי - מסתבר ש<strong>כל </strong>פונקציונל לינארי על {% equation %}V{% endequation %} ניתן לתיאור באופן יחיד באמצעות המכפלה הפנימית (במרחב אינסוף ממדי זה פשוט לא נכון, אם כי לא אציג דוגמאות נגדיות כעת).

בואו ראשית כל נשחק במשחק הבא: ניקח {% equation %}y\in V{% endequation %} כלשהו ונחשוב עליו כעל "קבוע". אז אפשר להגדיר פונקציה {% equation %}f_{y}:V\to\mathbb{C}{% endequation %} של "מכפלה פנימית ב-{% equation %}y{% endequation %}", כלומר {% equation %}f_{y}\left(x\right)=\left\langle x,y\right\rangle {% endequation %}. קל לראות ש-{% equation %}f_{y}{% endequation %} היא פונקציונל לינארי, פשוט בגלל הלינאריות של המכפלה הפנימית; מה שקורה במרחב סוף ממדי הוא ש<strong>כל</strong> פונקציונל לינארי {% equation %}f{% endequation %} ניתן להצגה כ-{% equation %}f_{y}{% endequation %} עבור {% equation %}y{% endequation %} אחד ויחיד ספציפי. איך מוצאים אותו? בעזרת הכלי הסטנדרטי שלנו להתמודדות עם מרחבי מכפלה פנימית - בסיסים אורתונורמליים.

ניקח בסיס אורתונורמלי {% equation %}u_{1},\dots,u_{n}{% endequation %} למרחב. בהינתן {% equation %}x\in V{% endequation %} כלשהו, אנחנו יודעים ש-{% equation %}x=\sum\left\langle x,u_{i}\right\rangle u_{i}{% endequation %} (זוכרים? כשמייצגים איבר בבסיס אורתונורמלי, המקדם של איבר הבסיס {% equation %}u_{i}{% endequation %} בצירוף הלינארי הוא בדיוק המכפלה הפנימית של {% equation %}x{% endequation %} ב-{% equation %}u_{i}{% endequation %}. קוראים לזה "מקדם פורייה" של {% equation %}x{% endequation %} בבסיס {% equation %}u_{1},\dots,u_{n}{% endequation %}). אנחנו גם יודעים איך מכפלה פנימית של שני איברים כלליים נראית: אם {% equation %}y=\sum\left\langle y,u_{i}\right\rangle u_{i}{% endequation %} הוא איבר תמים אחר של {% equation %}V{% endequation %} ואנו כופלים אותו ב-{% equation %}x{% endequation %}, נקבל:

{% equation %}\left\langle x,y\right\rangle =\left\langle \sum\left\langle x,u_{i}\right\rangle u_{i},\sum\left\langle y,u_{j}\right\rangle u_{j}\right\rangle =\sum_{i,j}\left\langle x,u_{i}\right\rangle \overline{\left\langle y,u_{j}\right\rangle }\left\langle u_{i},u_{j}\right\rangle =\sum_{i}\left\langle x,u_{i}\right\rangle \overline{\left\langle y,u_{i}\right\rangle }{% endequation %}

כעת, אם נפעיל את {% equation %}f{% endequation %} על {% equation %}x{% endequation %} ונשתמש בלינאריות שלו, נקבל ש-{% equation %}f\left(x\right)=\sum\left\langle x,u_{i}\right\rangle f\left(u_{i}\right){% endequation %}. זה מאוד מזכיר את המשוואה שלמעלה, אם {% equation %}f\left(u_{i}\right)=\overline{\left\langle y,u_{i}\right\rangle }{% endequation %}, ובניסוח אחר {% equation %}\left\langle y,u_{i}\right\rangle =\overline{f\left(u_{i}\right)}{% endequation %}. זה מוביל אותנו ל<strong>הגדרה</strong> הבאה: {% equation %}y=\sum\overline{f\left(u_{i}\right)}u_{i}{% endequation %}. כעת קל לוודא שאכן {% equation %}f\left(x\right)=\left\langle x,y\right\rangle {% endequation %} לכל {% equation %}x\in V{% endequation %}. כדי לוודא ש-{% equation %}y{% endequation %} הוא יחיד, נניח כי ישנם {% equation %}y_{1},y_{2}{% endequation %} כך ש-{% equation %}\left\langle x,y_{1}\right\rangle =f\left(x\right)=\left\langle x,y_{2}\right\rangle {% endequation %} לכל {% equation %}x{% endequation %}, כלומר {% equation %}\left\langle x,y_{1}-y_{2}\right\rangle =0{% endequation %} לכל {% equation %}x{% endequation %}. במרחב סוף ממדי זה יכול לקרות רק אם {% equation %}y_{1}-y_{2}=0{% endequation %}, כלומר {% equation %}y_{1}=y_{2}{% endequation %} (למה בעצם אם {% equation %}\left\langle x,y\right\rangle =0{% endequation %} לכל {% equation %}x{% endequation %} זה גורר ש-{% equation %}y=0{% endequation %}? תרגיל טוב).

<a href="http://www.gadial.net/2011/11/02/linear_functionals/">בפוסט שלי</a> על פונקציונלים לינאריים הערתי שיש איזומורפיזם בין {% equation %}V{% endequation %} ובין {% equation %}V^{*}{% endequation %}- מרחב הפונקציונלים הלינאריים מעל {% equation %}V{% endequation %}, אך הוא אינו "קנוני" - כלומר, עבור בחירות בסיס שונות עבור {% equation %}V{% endequation %} נקבל איזומורפיזמים שונים ואין דרך להכריע מי מביניהם יותר "נכון". במקרה שבו {% equation %}V{% endequation %} הוא מרחב מכפלה פנימית, קיבלנו איזומורפיזם קנוני לעילא ולעילא - {% equation %}y\mapsto f_{y}{% endequation %}, שאינו תלוי בבחירת בסיס זה או אחר למרחב. כמובן שאני קצת משקר כאן - האיזומורפיזם הזה עדיין תלוי בבחירת <strong>מכפלה פנימית</strong> למרחב, והיא שקולה לבחירת בסיס, אבל נעזוב את זה.

בואו נעבור לדבר עכשיו על משהו אחר - אופרטורים לינאריים, כלומר טרנספורמציות לינאריות {% equation %}T:V\to V{% endequation %} מ-{% equation %}V{% endequation %} לעצמו. השאלה הראשונה שאני שואל היא שאלה של סימטריה: נניח שיש לי וקטורים {% equation %}x,y{% endequation %}, ואני מפעיל את {% equation %}T{% endequation %} על {% equation %}x{% endequation %} ואז כופל את התוצאה עם {% equation %}y{% endequation %} - האם אקבל את אותו הדבר כאילו הפעלתי את {% equation %}T{% endequation %} על {% equation %}y{% endequation %} וכפלתי את {% equation %}x{% endequation %} בזה? כלומר, האם {% equation %}\left\langle Tx,y\right\rangle =\left\langle x,Ty\right\rangle {% endequation %}? התשובה היא שלא תמיד, ושאופרטורים שמקיימים את התכונה הזו הם מעניינים למדי - הם נקראים <strong>אופרטורים הרמיטיים</strong> (ומעל {% equation %}\mathbb{R}{% endequation %}- <strong>סימטריים</strong>). קחו רגע ונסו להסביר לעצמכם למה השוויון {% equation %}\left\langle Tx,y\right\rangle =\left\langle x,Ty\right\rangle {% endequation %} כלל לא מובן מאליו.

אוקיי, אז לא תמיד מתקיים {% equation %}\left\langle Tx,y\right\rangle =\left\langle x,Ty\right\rangle {% endequation %}. עדיין, התחושה האינטואיטיבית היא שמידת מה של סימטריה צריכה להיות כאן, והתחושה הזו מדוייקת - לכל אופרטור {% equation %}T{% endequation %} מותאם אופרטור {% equation %}T^{*}{% endequation %} - <strong>הצמוד ההרמיטי</strong> של {% equation %}T{% endequation %} - כך ש-{% equation %}\left\langle Tx,y\right\rangle =\left\langle x,T^{*}y\right\rangle {% endequation %}. זה המשפט העיקרי שאני רוצה להוכיח בפוסט. כעת אפשר לתאר אופרטור הרמיטי בצורה קצת יותר פשוטה: זה אופרטור שמקיים {% equation %}T=T^{*}{% endequation %}, כלומר הוא שווה לצמוד ההרמיטי שלו; כלומר הוא <strong>צמוד לעצמו</strong>. זה עוד שם לאופרטורים הרמיטיים.

הוכחת הקיום של {% equation %}T^{*}{% endequation %} היא מחוכמת ואלגנטית. הבה ונקבע את {% equation %}y{% endequation %} לרגע, אז אפשר להגדיר פונקציונל לינארי {% equation %}f\left(x\right)=\left\langle Tx,y\right\rangle {% endequation %}. במילים אחרות, בהינתן {% equation %}x{% endequation %} נתעלל בו על ידי הפעלת {% equation %}T{% endequation %} עליו ואז מכפלה פנימית של כל זה עם {% equation %}y{% endequation %}. זה אכן פונקציונל כי {% equation %}T{% endequation %} היא טרנספורמציה לינארית ומכפלה פנימית היא לינארית. אם {% equation %}f{% endequation %} הזו היא פונקציונל, אז יש איזה שהוא {% equation %}z{% endequation %} שמגדיר אותה: {% equation %}f\left(x\right)=\left\langle x,z\right\rangle {% endequation %}. במילים אחרות, {% equation %}\left\langle x,z\right\rangle =\left\langle Tx,y\right\rangle {% endequation %}, ולכן אם קיים בכלל {% equation %}T^{*}{% endequation %} שהוא צמוד הרמיטי של {% equation %}T{% endequation %}, אז הוא מקיים {% equation %}\left\langle x,z\right\rangle =\left\langle x,T^{*}y\right\rangle {% endequation %}. במילים אחרות, אנחנו נדחפים להגדרה {% equation %}T^{*}\left(y\right)=z{% endequation %}.

באופן הזה אפשר להגדיר את {% equation %}T^{*}{% endequation %} לכל איבר {% equation %}y\in V{% endequation %}. צריך לוודא שאכן קיבלנו אופרטור לינארי; שום דבר לא מבטיח לנו שהתנאי ההכרחי שמצאנו עבור {% equation %}T^{*}{% endequation %} (שקבע באופן יחיד איך הוא חייב להיראות) הוא גם מספיק כדי שהוא אכן יהיה אופרטור לינארי; זה ה"קסם" שיש כאן. אם כן, יהיו {% equation %}y_{1},y_{2}{% endequation %} כך ש-{% equation %}T^{*}\left(y_{1}\right)=z_{1}{% endequation %} ו-{% equation %}T^{*}\left(y_{2}\right)=z_{2}{% endequation %}; נרצה להראות ראשית ש-{% equation %}T^{*}\left(y_{1}+y_{2}\right)=z_{1}+z_{2}{% endequation %}. נסמן {% equation %}T^{*}\left(y_{1}+y_{2}\right)=z{% endequation %}; אנחנו יודעים ש-

{% equation %}\left\langle x,z\right\rangle =\left\langle Tx,y_{1}+y_{2}\right\rangle =\left\langle Tx,y_{1}\right\rangle +\left\langle Tx,y_{2}\right\rangle =\left\langle x,z_{1}\right\rangle +\left\langle x,z_{2}\right\rangle =\left\langle x,z_{1}+z_{2}\right\rangle {% endequation %}

השוויון {% equation %}\left\langle x,z\right\rangle =\left\langle x,z_{1}+z_{2}\right\rangle {% endequation %} עבור ערך ספציפי של {% equation %}x{% endequation %} לא מספיק כדי להסיק ש-{% equation %}z=z_{1}+z_{2}{% endequation %}, אבל מכיוון שזה נכון עבור כל {% equation %}x{% endequation %} זה מספיק (שוב - למה?). באופן דומה גם מראים ש-{% equation %}T^{*}\left(\lambda y\right)=\lambda T^{*}\left(y\right){% endequation %} ולכן {% equation %}T^{*}{% endequation %} אכן לינארית.

עכשיו, אנחנו יודעים שבמרחב סוף-ממדי כל טרנספורמציה לינארית ניתנת לייצוג באמצעות מטריצות; נניח שקבענו איזה בסיס <strong>אורתונורמלי </strong>של המרחב, {% equation %}B=\left\{ u_{1},\dots,u_{n}\right\} {% endequation %}, ואנחנו מסתכלים על {% equation %}\left[T\right]_{B}{% endequation %} - המטריצה המייצגת של {% equation %}T{% endequation %} ביחס לבסיס הזה. מה הקשר בינה ובין המטריצה המייצגת של {% equation %}\left[T^{*}\right]_{B}{% endequation %}? האם יש לנו תיאור קונקרטי וקל לחישוב של האופן שבו הייצוג הנוח של {% equation %}T{% endequation %} הופך לייצוג נוח של {% equation %}T^{*}{% endequation %}? התשובה חיובית, במקרה שבו {% equation %}B{% endequation %} אורתונורמלי. זכרו שהעמודה ה-{% equation %}i{% endequation %}-ית של המטריצה {% equation %}\left[T^{*}\right]_{B}{% endequation %} היא בדיוק וקטור הקואורדינטות לפי {% equation %}B{% endequation %} של {% equation %}T^{*}\left(u_{i}\right){% endequation %} (הפעלת {% equation %}T^{*}{% endequation %} על איבר הבסיס ה-{% equation %}i{% endequation %} ב-{% equation %}B{% endequation %}), או במילים אחרות - הכניסה ה-{% equation %}j,i{% endequation %} של {% equation %}\left[T^{*}\right]_{B}{% endequation %} היא בדיוק {% equation %}\left\langle T^{*}\left(u_{i}\right),u_{j}\right\rangle {% endequation %}. בואו נעשה תעלול אלגברי זריז:

{% equation %}\left\langle T^{*}\left(u_{i}\right),u_{j}\right\rangle =\overline{\left\langle u_{j},T^{*}\left(u_{i}\right)\right\rangle }=\overline{\left\langle T\left(u_{j}\right).u_{i}\right\rangle }{% endequation %}

אבל מה זה {% equation %}\left\langle T\left(u_{j}\right).u_{i}\right\rangle {% endequation %}? זה בדיוק המקדם של {% equation %}u_{i}{% endequation %} בצירוף הלינארי שמגדיר את {% equation %}T\left(u_{j}\right){% endequation %}. במילים אחרות, זו הכניסה ה-{% equation %}i,j{% endequation %} של המטריצה {% equation %}\left[T\right]_{B}{% endequation %}!

זה מוביל אותנו להגדרה הבאה: אם {% equation %}A{% endequation %} היא מטריצה ריבועית כלשהי מעל המרוכבים, אז <strong>המטריצה הצמודה</strong> שלה מוגדרת בתור {% equation %}A^{*}=\overline{A^{t}}{% endequation %}, כלומר ביצוע שחלוף והצמדה, כלומר {% equation %}A_{ij}^{*}=\overline{A_{ji}}{% endequation %}. כשמדובר במטריצות קל למדי לראות שמתקיימות התכונות הבאות של הצמדה (שאני מנסח בלשון טרנספורמציות כי הן תקפות באותה מידה גם לטרנספורמציות):
<ol>
	<li>{% equation %}\left(T+S\right)^{*}=T^{*}+S^{*}{% endequation %}</li>
	<li>{% equation %}\left(\lambda T\right)^{*}=\overline{\lambda}T^{*}{% endequation %}</li>
	<li>{% equation %}\left(TS\right)^{*}=S^{*}T^{*}{% endequation %}</li>
	<li>{% equation %}\left(T^{*}\right)^{*}=T{% endequation %}</li>
</ol>
ארבע התכונות הללו מראות לנו שיש מבנה אלגברי כלשהו לפעולה של "קח טרנספורמציה והחזר את הצמודה שלה" - בלשון פונקציות, {% equation %}f\left(T\right)=T^{*}{% endequation %}. תכונות 1 ו-2 אומרות שהפונקציה הזו היא "כמעט לינארית", עד כדי כך שהוצאת סקלר כרוכה בהצמדה שלו - פונקציה כזו נקראת <strong>אנטי-לינארית</strong>, או <strong>לינארית-צמודה</strong>. התכונה הבאה מראה ש-{% equation %}f{% endequation %} הופכת איברים בכפל: {% equation %}f\left(TS\right)=f\left(S\right)f\left(T\right){% endequation %}.
לפונקציה שמקיימת {% equation %}f\left(ST\right)=f\left(S\right)f\left(T\right){% endequation %} קוראים <strong>הומומורפיזם</strong> ולכן ההצמדה שלנו, שעושה מעין היפוך של זה, נקראת <strong>אנטי-הומומורפיזם</strong> (מאוד אנטי כל העסק - אנטי-לינארי, אנטי-הומומורפיזם...). לסיום, תכונה 4 אומרת ש-{% equation %}f\left(f\left(T\right)\right)=T{% endequation %} - לפונקציה שמקיימת את זה קוראים <strong>אינבולוציה</strong>. בקיצור, ההצמדה שלנו היא פעולה שמקיימת כמה תכונות מוכרות וחביבות על המתמטיקאים.

באופן לא מפתיע, מסתבר שאפשר להשתמש בתכונות הללו כבסיס להכללה; זה הבסיס לתחום במתמטיקה של <strong>אלגברות {% equation %}C{% endequation %}-כוכב </strong>(אלגבראות {% equation %}C^{*}{% endequation %}), שהן אלגבראות בנך עם פונקציה שמקיימת את תכונות 1-4 ועוד כמה תכונות שלא אכנס אליהן כעת (כמו שלא אכנס להגדרה של אלגברת בנך; כל הדברים הללו קשורים לאלגברה לינארית במרחב אינסוף ממדי). לא ארחיב לכיוון הזה עכשיו, אלא אדבר דווקא על דוגמה פשוטה ביותר לפונקציה לא טריוויאלית נוספת שמקיימת את תכונות 1-4, שאולי כבר קפצה לחלקכם לראש - הצמדה של מספרים מרוכבים. תכונות 1,2,4 שלה באות באופן טבעי לחלוטין; תכונה 3 נובעת מכך ש-{% equation %}\overline{zw}=\overline{z}\cdot\overline{w}{% endequation %} ומכך שבמספרים מרוכבים הכפל הוא קומוטטיבי, כלומר {% equation %}\overline{w}\cdot\overline{z}{% endequation %}. הדמיון הזה מסביר את השם <strong>צמוד</strong> בהקשר הנוכחי.

הדמיון לא נגמר כאן. במספרים מרוכבים מתקיימת התכונה הנחמדה שאם {% equation %}z{% endequation %} הוא מספר מרוכב כלשהו, אז {% equation %}\frac{z+\overline{z}}{2}{% endequation %} הוא החלק הממשי של {% equation %}z{% endequation %}, ואילו {% equation %}\frac{z-\overline{z}}{2}{% endequation %} הוא החלק המדומה של {% equation %}z{% endequation %} (בדקו בעצמכם אם אינכם זוכרים/יודעים/מאמינים). בואו נעשה דבר דומה עם טרנספורמציות: ניקח {% equation %}T{% endequation %} כלשהי ונגדיר {% equation %}T_{R}=\frac{T+T^{*}}{2}{% endequation %} ו-{% equation %}T_{I}=\frac{T-T^{*}}{2}{% endequation %}.

עכשיו, {% equation %}T_{R}^{*}=\frac{1}{2}\left(T+T^{*}\right)^{*}=\frac{T^{*}+\left(T^{*}\right)^{*}}{2}=\frac{T+T^{*}}{2}=T_{R}{% endequation %}, ולכן {% equation %}T_{R}{% endequation %} צמודה לעצמה; בדומה נקבל ש-{% equation %}T_{I}^{*}=-T_{I}{% endequation %} - לטרנספורמציה שמקיימת את התכונה הזו קוראים <strong>אנטי-הרמיטית</strong>. כפי שאתם ודאי שמים לב, {% equation %}T_{R}{% endequation %} מתנהג ביחס להצמדה כמו שמספר ממשי מתנהג ביחס אליה, ו-{% equation %}T_{I}{% endequation %} מתנהג כמו שמספר מדומה טהור מתנהג ביחס אליה. כעת שימו לב לכך ש-{% equation %}T=T_{R}+T_{I}{% endequation %} , כלומר כל טרנספורמציה לינארית במרחב סוף ממדי ניתן לרשום כסכום של טרנספורמציה הרמיטית וטרנספורמציה אנטי-הרמיטית (למעשה, אפשר גם להגדיר {% equation %}T_{I}^{\prime}=\frac{T-T^{*}}{2i}{% endequation %} ולקבל טרנספורמציה הרמיטית ואז הפירוק הוא {% equation %}T=T_{R}+iT_{I}^{\prime}{% endequation %} - שוב, בדיוק כמו עם מספרים מרוכבים). האנלוגיה הזו הלהיבה אותי מאוד בשעתו; זה תמיד מעניין לראות איך תכונות שקיימות במרוכבים הן ממש לא נחלתם הבלעדית אלא מעידות כנראה על משהו עמוק יותר מתחת לפני השטח.
